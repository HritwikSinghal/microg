# Privileged platform permissions -- AOSP API 30 (Android 11).
#
# WHAT THIS IS
# Each line is the name of a platform-defined permission whose protectionLevel
# contains "privileged" in the AOSP framework manifest for this API. These are
# the permissions a /system/priv-app may be granted via a privapp-permissions
# allowlist XML. A permission NOT in this set cannot be auto-granted to a
# privileged app no matter what the APK requests -- so genperms.py intersects the
# APK's requested permissions with this set (see design spec section 6.1).
#
# HOW TO REGENERATE (do this per target API on a microG/AOSP bump)
# The source of truth is the platform framework manifest, NOT any APK:
#   frameworks/base/core/res/AndroidManifest.xml   (in the AOSP tree for the API)
# A permission qualifies when its android:protectionLevel attribute contains the
# "privileged" flag, e.g.:
#   protectionLevel="signature|privileged"
#   protectionLevel="signature|privileged|development"
# Extract them (from a checked-out AOSP tree at the target tag) with something like:
#   xmllint --xpath \
#     '//permission[contains(@android:protectionLevel,"privileged")]/@android:name' \
#     frameworks/base/core/res/AndroidManifest.xml
# then strip the android:name=" wrapper and sort -u. genperms.py can also parse
# such a manifest directly via --aosp-manifest, which applies the same rule.
#
# SAFETY NOTE (design spec 6.1): under-listing causes a bootloop under enforce;
# over-listing is harmless. When unsure whether a perm is privileged at this API,
# KEEP it here -- prefer the superset. genperms.py unions multiple API files.
#
# This is a curated, documented subset covering the permissions microG's bundled
# components actually request; it is intentionally a superset where doubtful.
# Blank lines and lines starting with '#' are ignored.
