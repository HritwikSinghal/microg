# Phase 1 Handoff -- Module-only installer

This is a fresh-context brief for the agent picking up Phase 1. It is grounded in
the actual repo state as of 2026-06-20 (Phase 0 code-complete). Read it top to
bottom before writing code. Authoritative design: `docs/superpowers/specs/2026-06-20-microg-zygisk-installer-design.md`.

---

## 1. Mission

Build the on-device installer: a generic, data-driven `customize.sh` that reads
`components.conf` and overlays microG as privileged system apps under
`$MODPATH/system`, working across Magisk (magic-mount) and KernelSU / APatch
(OverlayFS). Phase 1 is MODULE MODE ONLY -- it never writes real partitions, so
free-space / dynamic-partition / A-B concerns are out of scope (Phase 3).

## 2. Project rules (non-negotiable)

- **The agent does NOT build, flash, or run on a device.** The user does that.
  Write code + host tests; do not attempt to assemble a real ZIP or push to a
  phone. (Host-side shellcheck/BATS is fine and expected.)
- **Public repo.** Never commit home paths, private directory names, the user's
  work email, or device codenames. Use generic placeholders.
- **No signature spoofing** anywhere in the installer. It is shipped separately as
  a guide (Phase 5).
- **ASCII-only** in all files and output.
- **The CI permission invariant is the bootloop cure** -- do not weaken it. Perms
  XMLs are generated at build time and gated; on-device scripts only SELECT and
  PLACE pre-generated XML, never generate.
- Update `claude/progress.md` after each completed task (mark `[x]`, recount, bump
  the date). It is the primary tracker.

## 4. Hard contracts the new code MUST honor

### 4.1 `components.conf` (the install-time input)

Generated by `lib/manifest.py emit-conf`. Whitespace-separated, one row per
non-deferred component, parseable with shell `read`. Deferred components (Phonesky,
empty url) are intentionally ABSENT. Column order is fixed (7 columns):

```
# name       pkg                      asset               partition  type       perms                conflicts
GmsCore    com.google.android.gms   apks/GmsCore.apk    product    app        perms/gmscore.xml    -
GsfProxy   com.google.android.gsf   apks/GsfProxy.apk   product    app        perms/gsfproxy.xml   -
FakeStore  com.android.vending      apks/FakeStore.apk  product    app        perms/fakestore.xml  Phonesky
MapsV1     com.google.android.maps  apks/maps.jar       product    framework  perms/mapsv1.xml     -
```

- `asset` is the in-ZIP path (relative to `$MODPATH`): `apks/<Name>.apk` for app,
  `apks/maps.jar` for framework.
- `perms` is the in-ZIP path, **lowercased** filename: `perms/<name.lower()>.xml`.
  This lowercasing is a contract shared by manifest.py, genperms.py, and
  invariant.py -- match it exactly.
- `conflicts` is comma-separated component names, or `-` when none.
- `type` is the placement discriminator: `app` vs `framework`.
- A `sysconfig-microg.xml` is also emitted into `perms/` (behavioral exemptions);
  it is NOT a per-component row -- handle it separately in perms.sh.

To regenerate locally for a test fixture:
`python3 lib/manifest.py emit-conf --out /tmp/components.conf`

### 4.2 `common/detect.sh` (already done -- call these)

No exported variables; every probe ECHOES a value, capture with `$(...)`:

- `detect_api` -> integer, or `0` if unknown
- `detect_is64bit` -> `true` | `false`
- `detect_arch` -> `arm64` | `arm` | `x86_64` | `x86` | `unknown`
- `detect_root_manager` -> `magisk` | `kernelsu` | `apatch` | `unknown`
- `detect_mount_engine` -> `overlayfs` | `magic-mount` | `unknown`
- `detect_partitions` -> space-separated, canonical order, e.g. `system product system_ext`

Mocking seams (set in tests, never in prod): `$GETPROP`, `$DETECT_ROOT`,
`$DETECT_MOUNTS`, `$DETECT_PART_CANDIDATES`.

### 4.3 `common/log.sh` (already done -- call these)

- `log_init` once before logging (idempotent; creates the log dir).
- `log_info` / `log_warn` / `log_error` / `log_debug` (debug gated on `$MICROG_DEBUG`).
- Writes `YYYY-MM-DD HH:MM:SS [LEVEL] msg` to `$MICROG_LOG_DIR/$MICROG_LOG_FILE`
  (default `/data/adb/microg_installer/selfcheck.log`), mirrors to `ui_print` if
  defined else stderr. Degrades quietly. Override `$MICROG_LOG_DIR` in tests.

### 4.4 Magisk install environment (exported into `customize.sh`)

`$MODPATH`, `$API`, `$ARCH`/`$ABI`, `$IS64BIT`, `$BOOTMODE`, and presence markers
`$KSU`/`$KSU_VER`, `$APATCH`/`$APATCH_VER`, `$MAGISK_VER`/`$MAGISK_VER_CODE`.
`ui_print` is defined by update-binary. There is NO generic `abort` helper from the
vendored binary -- if you need to fail, `log_error` then `exit 1` (or define a small
local abort). Prefer `detect_api` over raw `$API` when you need the safe `0` fallback.

## 5. On-device placement layout (Phase 1 target)

Everything lands under `$MODPATH/system` (writable /data; magic-mount and OverlayFS
both overlay it):

- app:       `$MODPATH/system/<partition>/priv-app/<Name>/<Name>.apk`
- framework: `$MODPATH/system/<partition>/framework/<asset basename>` (Phase 2 path,
  but design it now) + its permissions XML in `<partition>/etc/permissions/`
- privapp allowlist XML: `$MODPATH/system/<partition>/etc/permissions/<file>.xml`
- sysconfig XML:         `$MODPATH/system/<partition>/etc/sysconfig/sysconfig-microg.xml`

`<partition>` comes from the `partition` column (currently `product` for all). It is
only a path prefix inside the module in Phase 1. Pick the partition that actually
exists on the device when possible (cross-check `detect_partitions`); fall back to
the declared one with a `log_warn`.

## 6. Task breakdown (7 units -- the Phase 1 checklist in progress.md)

Implement in this order; each is independently host-testable with BATS.

1. **`customize.sh` manifest interpreter.** Source `common/log.sh` + `detect.sh` +
   `place.sh` + `perms.sh` + `cleanup.sh`. `log_init`, probe device, log a banner.
   Read `components.conf` line by line (skip `#`/blank), and for each row:
   resolve conflicts (section 6/4.1), place the asset (place.sh), place its perms
   XML (perms.sh). Idempotent on re-flash (replace stale files, never merge).
   Guard edge cases from spec section 7 (arch mismatch, API < 26 harmless, real GMS
   present).

2. **`common/place.sh` -- ModuleOverlayPlacer.** A `place_app` / `place_framework`
   that copies `$MODPATH/<asset>` into the overlay tree (section 5) and removes the
   staging copy if appropriate. Keep a `Placer`-style seam so Phase 3
   DirectPartitionPlacer can slot in. No real-partition writes.

3. **`common/perms.sh` -- XML selection + placement by API level.** Select the
   correct pre-generated XML from `perms/` (Phase 1 has one variant; design the
   selector for multiple API variants), validate it parses, place into the matching
   partition's `etc/permissions` (privapp) and `etc/sysconfig` (sysconfig-microg).
   Never generate XML on device.

4. **`common/cleanup.sh` -- StockGmsRemover via declarative REPLACE sentinel.**
   Prefer Magisk `REPLACE` over `mknod` whiteouts (silently no-op under OverlayFS).
   Detect-engine-aware (use `detect_mount_engine`). Removal that needs early boot
   goes through post-fs-data.sh.

5. **`post-fs-data.sh` + `service.sh`.** post-fs-data: engine-aware stock-GMS
   removal, runs pre-decryption -- never touch `/data/data` there (FBE). service.sh:
   boot-gated (wait on `sys.boot_completed`) self-check that writes to selfcheck.log
   only; no placement work.

6. **selfcheck.log diagnostics.** Use log.sh; record detected env, what was placed,
   conflicts resolved, and a final OK/PROBLEM verdict for the user to read.

7. **shellcheck clean** across all scripts (the CI `checks` job runs
   `shellcheck` over the script list in `.github/workflows/build.yml` -- add the new
   scripts to that list).

## 7. Testing requirements

- Follow the existing BATS pattern in `test/detect.bats` / `test/log.bats` and
  `test/test_helper.bash` (helpers `getprop_fixture`, `make_fixture_root`,
  `write_mounts`; fixtures under `test/fixtures/`).
- Mock the device: set `$MODPATH` to a tmpdir, `$DETECT_ROOT`/`$DETECT_MOUNTS` to
  fixtures, `$MICROG_LOG_DIR` to a tmpdir. Never write to a real path.
- Write a `components.conf` fixture (or generate via manifest.py) and assert the
  interpreter places the expected files under the fake `$MODPATH/system/...` tree.
- Cover: app vs framework placement, conflict resolution (FakeStore vs Phonesky),
  re-flash idempotence, missing-partition fallback, perms XML placement paths.
- Add new BATS files to the `bats test/` run (already globbed) and the new scripts
  to the shellcheck list in `build.yml`.
