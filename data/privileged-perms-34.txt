# WHAT THIS IS / HOW TO REGENERATE / SAFETY NOTE
# See the long header in data/privileged-perms-30.txt -- the meaning, the
# regeneration procedure (parse frameworks/base/core/res/AndroidManifest.xml for
# protectionLevel containing "privileged"), and the under-list-bootlooping safety
# rule are identical. This file holds the API 34 view; genperms.py unions a
# couple of API levels (e.g. --api 34 also reads the 30 file when present) so a
# perm that was privileged on an older platform stays grantable on devices still
# running it. Prefer the superset when unsure.
